Description

A <B>tree is an ordered collection of read-only <PB>key, and an optional <PB>value to go with them. This can be a map or set, but in general, it can have identical keys, (a multi-map). Internally, this is a B-tree, described in Bayer, McCreight, 1972 Large.

Advances it to the next element.

Parameter: TREE_NAME, TREE_KEY
<B> that satisfies C naming conventions when mangled, required, and an integral type, <PB>key, whose default is unsigned int. <PB> is private, whose names are prefixed in a manner to avoid collisions.
Parameter: TREE_VALUE
TRIE_VALUE is an optional payload to go with the type, <PB>value.
Parameter: TREE_COMPARE
A function satisfying <PB>compare_fn. Defaults to ascending order. Required if TREE_KEY is changed to an incomparable type.
Parameter: TREE_EXPECT_TRAIT
Do not un-define certain variables for subsequent inclusion in a parameterized trait.
Parameter: TREE_TO_STRING_NAME, TREE_TO_STRING
To string trait contained in to_string.h; an optional unique <SZ> that satisfies C naming conventions when mangled and function implementing <PSZ>to_string_fn.
Standard:
C89

Typedef Aliases

<PB>key

typedef TREE_KEY <PB>key;

A comparable type, defaults to unsigned. Note that key is used loosely; there can be multiple keys with the same value stored in the same <B>tree, if one chooses.

<PB>value

typedef TREE_VALUE <PB>value;

On TREE_VALUE, otherwise just a (multi)-set of <PB>key.

<PB>compare_fn

typedef int(*<PB>compare_fn)(<PB>key_c a, <PB>key_c b);

Returns a positive result if a is out-of-order with respect to b, inducing a strict weak order. This is compatible, but less strict then the comparators from bsearch and qsort; it only needs to divide entries into two instead of three categories.

<PB>entry

typedef struct <B>tree_entry <PB>entry;

On TREE_VALUE, otherwise it's just an alias for pointer-to-<PB>key.

<PSTR>to_string_fn

typedef void(*<PSTR>to_string_fn)(<PSTR>element_c, char(*)[12]);

src/to_string.h: responsible for turning the argument into a 12-char null-terminated output string.

Struct, Union, and Enum Definitions

<B>tree_entry

struct <B>tree_entry { <PB>key *x; <PB>value *value; };

On TREE_VALUE, creates a map from pointer-to-<PB>key to pointer-to-<PB>value. The reason these are pointers is because it is not connected in memory.

<B>tree

struct <B>tree;

To initialize it to an idle state, see <B>tree, TRIE_IDLE, {0} (C99), or being static.

<B>tree_iterator

struct <B>tree_iterator { struct <PB>iterator _; };

Stores an iteration in a tree. Generally, changes in the topology of the tree invalidate it.
